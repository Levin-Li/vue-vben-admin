# Admin Frontend AGENTS.md

<INSTRUCTIONS>
本文件适用于`frontend/admin/`目录及其所有子目录。处理本前端工程内的页面、组件、接口封装、状态、样式、构建脚本、Mock、文档或浏览器验证任务时,必须读取并遵循本文件列出的规则。

## 必读规则

本文件只是`frontend/admin/`规则入口,不能替代被引用的规则文件;执行任务前必须实际打开并读取对应规则文件.

- 仓库通用规则入口: `../../00-项目规则索引.md`
- 前端项目规则: `前端项目开发规则.md`

## 按任务类型读取

- 页面开发、布局、组件、主题和CRUD页面: `前端页面开发规则.md`
- API封装、接口请求、服务层调用和后端接口对接: `前端API开发规则.md`
- 页面操作、按钮、动作归属和交互流程: `前端页面操作规则.md`
- 表单校验、字段校验和提交前校验: `前端页面校验规则.md`
- 上传组件、文件选择和资源上传: `前端页面上传规则.md`
- 部署代理、开发代理和接口转发: `前端部署代理规则.md`

## 规则执行摘要

- 在已遵循仓库根目录和 `frontend/` 上层规则入口的前提下，本工程内部读取顺序为: 本文件 -> `前端项目开发规则.md` -> 当前任务对应专项规则;涉及多个主题时必须同时读取对应专项规则.
- `前端项目开发规则.md`负责总原则、入口和跨专项规则;同一细节在总规则和专项规则中都出现时,以专项规则为准. 该专项优先级只用于前端规则文档之间的冲突消解,不覆盖后端源码、运行时信息和规则文档之间的证据优先级.
- 前端页面规则用于指导显式页面开发,不允许把控制器元数据、OpenAPI或字段名猜测实现成运行时代码生成器、元数据生成器或通用页面自动拼装系统;共享组件只能复用稳定局部能力.
- 业务按钮必须来源于有效控制器方法及其`@CRUD.Op`/`@ResAuthorize`等元数据;刷新、全屏、列设置、导出等框架级表格工具按钮不属于业务按钮.
- 阻断式字段校验优先以Java Validation、后端DTO注解、JsonSchema或明确业务规则为准;字段名或类型语义只能决定输入体验、非阻断提示和组件选择.
- 页面视觉、交互、路由、下载和浏览器现场问题优先用当前in-app browser验证;如果in-app browser不可用,可按`前端项目开发规则.md`使用补充验证,但必须明确标注不能代表用户当前浏览器现场.
- 多`@CRUD.ListTable`操作归属字段统一使用`opRefTargetListName`;未显式填写按`default`处理,不再引入其它旧字段名.

## 执行要求

- 涉及本前端工程内代码、文档、页面、公共组件、构建脚本、Mock、规则文件或测试的新增需求、行为调整、项目定位变更、约束补充、任务追加或验收标准变化时,必须先遵循仓库根目录 OpenSpec 前置要求,创建或更新对应 `openspec/changes/<change-name>/` 和 `tasks.md`,再开始实现.
- 规则文件中尽量不要写绝对路径;描述仓库内文件时优先使用相对路径或文件名.
- 涉及前端页面、浏览器页面、交互流程、视觉状态、下载行为、路由可访问性或用户当前页面问题的验证,优先使用用户当前的 in-app browser 进行验证;例外条件和报告口径以`前端项目开发规则.md`为准.
- 如果更深层子目录中存在`AGENTS.md`,更深层规则优先.

</INSTRUCTIONS>
