# 前端页面上传规则

本文件用于单独维护管理后台前端页面中的图片、文件上传规则。规则审核通过后，可作为《前端页面开发规则》的配套规范引用。

## 上传字段识别规则

- 新增/编辑表单中，字符串类型或字符串集合类型字段，如果语义表示文件路径、图片地址、图标、头像、Logo、附件、证书、PDF、文档等，应使用上传组件，不使用普通输入框。
- 图片类字段包括但不限于：`avatar`、`icon`、`image`、`images`、`logo`、`photo`、`picture`、`pic`、`qr`，以及中文标题/描述包含“图片、头像、图标、Logo、二维码”等语义的字段。
- 文件类字段包括但不限于：`file`、`attachment`、`cert`、`certificate`、`doc`、`excel`、`pdf`，以及中文标题/描述包含“文件、附件、证书、文档、PDF”等语义的字段。
- 普通业务路径字段，例如路由 `path`、普通 URL、API 路径，不应因为字段名包含 `path/url` 就自动当作文件上传字段，除非标题/描述明确表示文件或图片资源。

## 单值与集合规则

- Java 字段类型不是集合时，上传组件只能保留一个文件或图片。
- 单值字段已有文件或图片后，不应再显示第二个新增上传卡片。
- Java 字段类型是集合时，上传组件必须支持多文件或多图片。
- 集合字段提交时保持数组结构，不应转成逗号字符串。
- 多图上传时，每张已有图片应作为独立图片卡片展示，并保留新增上传卡片。

## 服务端接口规则

- 所有图片和文件上传必须通过 `FileStorageController`，不要自行定义上传接口。
- 单文件上传接口使用：`POST /com.levin.oak.base/V1/api/fss/uploadSingleFile`。
- 多文件上传接口可使用：`POST /com.levin.oak.base/V1/api/fss/uploadFiles`。
- AMIS 兼容上传可使用：`POST /com.levin.oak.base/V1/api/fss/uploadSingleFile?amisUpload=true`。
- 本地 Vue 页面默认非 AMIS 上传时，后端 `uploadSingleFile` 返回 `ApiResp.ok(url)`；前端请求层解包后应按字符串 URL 处理。
- 如果上传接口没有返回字符串 URL，应视为接口异常并提示，不应靠前端猜测返回结构。

## 回显与预览规则

- 上传成功后，应立即把服务端返回的 URL 写回表单字段，并更新上传组件 fileList。
- 图片字段已有 URL 时，应在上传组件中回显缩略图。
- 点击已有图片卡片时，应打开大图预览。
- 点击已有非图片文件时，应打开文件 URL 或提供下载/查看入口。
- 如果服务端返回相对路径，例如 `/lfs/...`，前端开发服务器和 nginx 必须正确代理该路径到后端，确保图片可回显。
- 前端 dev server 代理路径应覆盖与后端资源相关的路径，例如 `/lfs`、`/com.levin.oak.base`、`/admin`、`/api`、`/api-docs`、`/swagger`、`/.well-known/acme-challenge` 等。

## 图片裁剪规则

- 图片类字段默认支持上传前裁剪。
- 单图片上传默认应弹出裁剪框，裁剪后再调用上传接口。
- 多图片上传一般不强制每张都裁剪；如页面有明确要求，可在上传每张图片前裁剪。
- 如果字段或页面明确声明不裁剪，可以关闭裁剪。
- 裁剪后的图片仍必须走统一上传接口。

## 粘贴截图上传规则

- 图片上传组件应支持截图粘贴上传。
- 鼠标悬停到图片上传卡片时，显示提示：`按 Ctrl/⌘ + V 粘贴截图到这`。
- 粘贴提示应显示在鼠标附近，不应固定显示在上传框内部。
- 鼠标移出上传组件区域后，不应继续接管粘贴事件。
- 鼠标悬停在新增图片上传卡片上粘贴截图时，应上传并新增图片。
- 鼠标悬停在已有图片卡片上粘贴截图时，应上传并替换当前图片。
- 多图上传场景下，每张已有图片卡片和新增上传卡片都应有明确的粘贴目标，不能把所有粘贴都追加到最后。
- 普通输入框、文本域等区域的粘贴文本行为不能被图片上传逻辑干扰。
