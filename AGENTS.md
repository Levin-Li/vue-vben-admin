# Admin Frontend AGENTS.md

<INSTRUCTIONS>
本文件适用于`frontend/admin/`目录及其所有子目录。处理本前端工程内的页面、组件、接口封装、状态、样式、构建脚本、Mock、文档或浏览器验证任务时,必须读取并遵循本文件列出的规则。

## 必读规则

本文件只是`frontend/admin/`规则入口,不能替代被引用的规则文件;执行任务前必须实际打开并读取对应规则文件.

- 仓库通用规则入口: `../../00-项目规则索引.md`
- 前端项目规则: `前端项目开发规则.md`

## 按任务类型读取

- 页面开发、布局、组件、主题和CRUD页面: `前端页面开发规则.md`
- API封装、接口请求、服务层调用和后端接口对接: `前端API开发规则.md`
- 页面操作、按钮、动作归属和交互流程: `前端页面操作规则.md`
- 表单校验、字段校验和提交前校验: `前端页面校验规则.md`
- 上传组件、文件选择和资源上传: `前端页面上传规则.md`
- 部署代理、开发代理和接口转发: `前端部署代理规则.md`

## 执行要求

- 规则文件中尽量不要写绝对路径;描述仓库内文件时优先使用相对路径或文件名.
- 涉及前端页面、浏览器页面、交互流程、视觉状态、下载行为、路由可访问性或用户当前页面问题的验证,必须使用用户当前的 in-app browser 进行验证.
- 如果更深层子目录中存在`AGENTS.md`,更深层规则优先.
