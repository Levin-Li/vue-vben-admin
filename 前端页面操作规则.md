# 前端页面操作规则

本文件用于约束管理后台页面中的菜单、列表页、操作按钮、`CRUD.Action`、显隐条件和权限处理。规则用于指导具体页面开发，不用于构建页面代码生成器。

## 一、页面和菜单来源

1. 页面和菜单必须来源于有效 Spring MVC 控制器及其注解元数据。

2. 控制器类或其父类上有 `@CRUD` 注解，表示该控制器可以对应一个菜单模块或业务页面模块。

3. 模块名称、业务名称优先取控制器上的 `@CRUD.title`、`@Tag.name`、`@CRUD.desc`、`@Tag.description` 等元数据；不要在前端重新发明名称。

4. 前端开发时必须同时检查业务控制器和父类控制器。
   业务控制器重写方法的优先级高于父类同语义方法；未重写时继承父类方法上的页面和按钮定义。

5. `@DisableApiOperation` 禁止的接口，不得作为前端按钮、菜单动作或批量动作暴露出来。
   这条规则同时适用于当前类和继承链上的方法。

## 二、列表页来源

1. 控制器方法上有 `@CRUD.ListTable` 注解，表示该方法对应一个列表页面。

2. 一个控制器下可以存在多个列表页面，前端不得假设一个模块只能有一个列表页。

3. 列表页唯一标识使用 `@CRUD.ListTable.name`；未显式填写时按默认值 `default` 处理。

4. 列表页标题、说明优先级：
   - `@CRUD.ListTable.title`
   - `@CRUD.ListTable.refEntityClass` 指定类型上的 `@Schema.title`
   - `@Operation.summary`
   - 方法语义名
   - 列表说明优先取 `@CRUD.ListTable.desc`，为空时可取 `refEntityClass.@Schema.description` 或 `@Operation.description`

5. `@CRUD.ListTable.visibleOn` 是列表页显示条件。
   前端应把它作为后端定义的显示规则处理，不得用页面硬编码替代。

## 三、按钮操作来源

1. 控制器方法上有 `@CRUD.Op` 注解，表示该方法对应一个前端操作按钮。

2. 前端按钮必须来源于后端控制器方法，不允许脱离控制器元数据自行定义业务按钮。

3. 按钮基础信息优先级：
   - 按钮名称：`@CRUD.Op.label`，为空时取 `@CRUD.Op.name`，再为空取 `@Operation.summary`
   - 按钮说明：`@CRUD.Op.desc`，为空时取 `@Operation.description`
   - 按钮图标：`@CRUD.Op.icon`
   - 按钮等级：`@CRUD.Op.level`

4. `@CRUD.Op.opRefTargetListName` 表示按钮属于哪个列表页。
   前端必须把按钮挂到对应列表页；未显式填写时按默认值 `default` 处理。
   旧属性名 `opRefTargetName` 不再作为规则依据。

5. `@CRUD.Op.opRefTargetType` 决定按钮显示位置：
   - `None`：页面级按钮，展示在列表页顶部工具栏。
   - `SingleRow`：单条记录按钮，展示在表格行操作区。
   - `MultipleRow`：批量按钮，展示在列表工具栏，并要求用户先选择多条记录。
   - `ListTable`：整个列表级按钮，展示在列表工具栏，不要求选择单条或多条记录。

6. `opRefTargetListName` 未显式填写时按默认值 `default` 处理。
   如果后端显式设置为空字符串，表示操作不绑定某个具体列表；本地页面应按页面结构放到页面级或默认列表工具区，不能把它解释为旧字段名。

7. 面向列表的操作，如新增、批量删除、批量更新，默认放在列表容器左上方。

8. 查询表单操作，如查询、重置、更多/收起筛选，默认放在查询表单容器右侧。

## 四、按钮显隐和权限

1. `@CRUD.Op.visibleOn` 是按钮显示条件，前端必须按后端表达式语义处理。

2. `visibleOn` 是业务显示条件，不是权限条件。

3. 按钮是否展示必须同时满足：
   - 接口未被 `@DisableApiOperation` 禁用。
   - 当前列表页匹配 `opRefTargetListName`。
   - 当前上下文满足 `opRefTargetType`。
   - `visibleOn` 条件为真。
   - 当前用户拥有该 API 方法对应权限。

4. `visibleOn` 支持的变量以后端注解说明为准。
   例如可使用 `__user` 表示当前登录用户；如果操作关联单条记录，还应支持当前记录的所有属性。

5. 页面和按钮权限应优先读取 API 方法上的 `@ResAuthorize` 元数据。
   不应在页面内散写权限表达式或按钮内联权限字符串。

6. 当前用户权限列表必须通过 `RbacController.authorizedPermissionList()` 获取。
   登录成功后和浏览器整页刷新后加载；普通路由切换不应重复加载。

7. 前端权限匹配必须使用后端提供的 `RbacPermissionMatchUtils.js/ts` 同等规则。
   不允许使用简单字符串相等判断替代。

8. `@CRUD.Op.confirmText` 表示操作确认提示内容。
   有值时必须展示确认框；特殊值 `None` 表示显式不弹确认框；为空时按“后端未要求确认”处理。

9. `@CRUD.Op.confirmTitle` 表示确认弹窗标题，有值时必须优先使用。

## 五、按钮行为来源

1. 按钮行为必须以控制器方法为准，优先读取：
   - 请求路径
   - HTTP 方法
   - `@Operation.summary`
   - `@Operation.description`
   - `@CRUD.Op`
   - `@CRUD.ListTable`
   - `@DisableApiOperation`
   - `@ResAuthorize`

2. `@CRUD.Op.action` 表示按钮点击时的主动作类型。

3. `@CRUD.Op.successAction` 表示接口成功后的后续动作。

4. `@CRUD.Op.failAction` 表示接口失败后的后续动作。

5. `@CRUD.Op.viewContainerType` 用于界面型动作的容器类型选择，常见值包括 `Dialog`、`Drawer`、`Inline`、`Auto`。

6. `@CRUD.Op.actionData` 表示调用动作时提交的数据来源。

7. `@CRUD.Op.resultActionData` 表示后续动作消费的数据来源。
   如果配置了 `resultActionData`，`successAction` / `failAction` 应优先使用提取后的结果数据；未配置时默认使用接口返回的 `data`。

8. 前端不能因为当前页面暂时没有实现某种动作容器，就私自改成别的交互。
   例如后端要求 `Link` 时，不能擅自改成 `Ajax`。

## 六、CRUD.Action 规则

1. 同一个 `CRUD.Action` 枚举值，在 `action`、`successAction`、`failAction` 中都可能出现，但语义必须结合当前阶段解释。

2. `action = Auto` 时，默认按 `Ajax` 处理。

3. `successAction = Auto` 时：
   - 如果主动作是 `Ajax`，默认按 `ReloadDataList` 处理。
   - 如果方法语义明确是查看详情或返回结果对象用于展示，应按后端明确配置的展示动作处理，不应继续按 `ReloadDataList` 兜底。

4. `failAction = Auto` 时，默认按统一错误提示处理，不额外做跳转或容器动作。

### 展示型动作

5. `ShowForm`
   - 用于展示接口成功返回结果中的对象详情。
   - 前端应基于响应结果中的 `data` 类型生成只读详情表单。
   - 不能复用当前列表页的创建表单、编辑表单或普通详情页表单。
   - 默认只读，不带保存提交。
   - 应遵守详情表单规则：列宽一致、最大宽度不超过页面内容区域的 70%、长内容独占一行。
   - 如果返回的 `data` 不是对象，应按返回值语义降级为合适的只读展示方式。

6. `ShowQrCode`
   - 用于把接口成功返回的数据按二维码内容处理。
   - 返回字符串时默认按二维码原始内容生成二维码。
   - 返回对象时优先提取明确的二维码字段，例如 `qrCode`、`mfaQrCode`、`url`、`value`。
   - 主体是二维码，不是详情表单。

7. `ShowImage`
   - 用于把接口成功返回的数据按图片资源处理。
   - 返回字符串时默认按图片地址、图片 data URL、base64 图片内容处理。
   - 返回对象时优先提取图片字段，例如 `image`、`imageUrl`、`url`、`src`、`coverUrl`。
   - 主体是图片预览，不是详情表单。

8. `ShowVideo`
   - 用于把接口成功返回的数据按视频资源处理。
   - 返回字符串时默认按视频地址处理。
   - 返回对象时优先提取视频字段，例如 `video`、`videoUrl`、`url`、`src`。
   - 主体是视频播放器或视频预览，不是详情表单。

9. `ShowIFrame`
   - 用于把接口成功返回的数据按可 iframe 加载的页面地址处理。
   - 返回字符串时默认按 iframe 的 `src` 地址处理。
   - 返回对象时优先提取 iframe 地址字段，例如 `url`、`src`、`iframeUrl`、`link`。
   - 主体是 iframe 页面，不是详情表单。

10. `ShowSchema`
    - 用于按 schema 或结构化数据展示结果。
    - 适用于结构化只读展示，不应默认等同于编辑表单。

### 请求型动作

11. `Ajax` 表示调用后端接口，是最常见的业务按钮动作。

12. `Jsonp` 只有控制器明确这样定义时才使用，前端不能自行把普通接口当 JSONP 处理。

13. `Js` 只有后端明确给出脚本语义时才使用，前端不能自行脑补脚本行为。

### 导航型动作

14. `Link` 表示用新窗口或新标签打开目标链接。

15. `Url` 表示当前窗口跳转到目标地址。

16. `Prev` 表示返回上一步。

17. `Next` 表示进入下一步。

18. `Close` 表示关闭当前对话框、抽屉或当前容器页签，必须结合当前容器上下文处理。

19. `Cancel` 表示取消当前动作或关闭当前交互，语义偏中止，不等同于普通关闭。

### 通讯型动作

20. `Tel` 按电话号码动作处理，例如 `tel:` 跳转。

21. `Email` 按邮箱动作处理，例如 `mailto:` 跳转。

22. `Copy` 把结果数据复制到剪贴板；如果配置了 `resultActionData`，优先复制提取后的结果数据。

### 数据刷新型动作

23. `ReloadDataList` 表示重新加载当前列表页数据。

24. `UpdateData` 表示更新当前页面上下文数据，不是整页刷新。
    如果页面暂未支持局部更新，应保留扩展点，不能偷换成别的动作。

25. `Toast` 表示展示轻提示。
    提示内容优先取后端返回消息；没有消息时再取按钮或操作默认成功文案。

## 七、开发落地要求

1. 新增或调整系统页面、工具栏按钮、表格行按钮、批量按钮时，必须先检查对应控制器的注解定义。

2. 前端应通过共享组件或公共工具统一处理稳定公共能力，但共享层不能演变成页面代码生成器或通用页面自动拼装系统。

3. 共享层至少可以统一处理列表页分组、按钮分组、`opRefTargetType`、`opRefTargetListName`、`visibleOn`、确认框、动作分发、权限匹配等稳定能力。

4. 不允许在每个页面重复写一套按钮显示规则；应优先沉淀到共享组件或公共工具层。

5. 如果后端控制器注解无法完整表达某个前端动作，优先补充后端元数据，再实现前端；不要让前端长期依赖硬编码例外。
